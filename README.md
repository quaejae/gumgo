# 🔐 금고털이 진행 보조 웹앱

오프라인 소셜게임 "금고털이"의 사회자 진행을 보조하는 웹앱입니다.

- **앱1 (`app1.html`)** — 진행자/방 스마트폰(세로). 참여자 입력, 금화 수령(슬라이더), 엿보기, 강탈/몰수/결과 진행. **유일한 조작 기기.**
- **앱2 (`app2.html`)** — 태블릿→프로젝터(가로) 대시보드. 누적 점수판 + 입장순서 + 라운드 결과 표시. **출력 전용.**
- 두 기기는 **Firebase Realtime Database**로 실시간 동기화됩니다.

## 폴더 구조
```
금고털이/
├─ index.html        역할 선택 랜딩
├─ app1.html         진행자/방 앱 (스마트폰)
├─ app2.html         대시보드 (태블릿)
├─ css/style.css
└─ js/
   ├─ firebase-config.js   Firebase 설정 (good-friends-a4722)
   ├─ sync.js              Realtime DB 동기화 래퍼
   ├─ game.js              순수 게임 로직(밸런스·강탈·중복·정산)
   ├─ app1.js
   └─ app2.js
```

## 사용 방법
1. **대시보드**: 태블릿 브라우저에서 `app2.html` 열기 → 프로젝터 연결(가로).
2. **진행자 폰**: 스마트폰 브라우저에서 `app1.html` 열기(세로).
3. 앱1에서 참여자 이름 입력 → **순서 배정 → 시작** → 앱2에 순서 표시.
4. 폰을 방에 두고 참여자가 순서대로 **금화 수령**(슬라이더로 개수 선택).
5. 전원 완료 후 첫 참여자 **엿보기** → **강탈자 공개 → 강탈 → 중복 몰수 → 결과 발표 → 다음 라운드**.
6. 마지막 라운드 후 **🏆 최종 우승 발표**.

> 두 기기가 **같은 방 코드**를 써야 합니다. 기본값 `MAIN`. 여러 게임을 동시에 하려면 주소 뒤에 `?room=코드`(예: `app1.html?room=PARTY2`)를 붙여 앱1·앱2 동일하게 맞추세요.

## 배포 (GitHub Pages)
1. 이 폴더 내용을 GitHub 저장소에 push.
2. 저장소 **Settings → Pages** → Branch를 `main` / 루트로 설정 → 저장.
3. 발급된 URL로 접속 (예: `https://아이디.github.io/저장소/app1.html`).

## 게임 밸런스 (인원수 자동)
- 인당 상한 `C = round(2.5 × 인원)`, 총 금화 `T = round(0.75 × C × 인원)` (설정 화면에서 수동 수정 가능).
- 예: 4명 → 30/10, 8명 → 120/20(룰북 기준), 12명 → 270/30.

## 확정 규칙
- **강탈 실패**: 강탈자가 그 라운드 보유한 금화 전부(강탈로 얻은 것 포함) 몰수.
- **중복 제거**: 원래 가져온 개수 기준 동률 몰수. 강탈로 변동된 사람(강탈자·피강탈자)과 0개는 제외.
- **강탈자 선정**: 최소 획득자(동률이면 다음으로 적은 단독자). 전원 동률이면 강탈 없음.
- **엿보기**: 라운드당 첫 순서 참여자 1명, 원하는 시점 1구간 확인.

## ⚠️ Firebase 보안 규칙
현재 Realtime Database가 **테스트 모드**라면 약 30일 후 접근이 잠깁니다. 실제 운영 전
콘솔 → Realtime Database → 규칙에서 아래처럼 `rooms` 경로만 열어두는 것을 권장합니다.
```json
{ "rules": { "rooms": { ".read": true, ".write": true } } }
```
